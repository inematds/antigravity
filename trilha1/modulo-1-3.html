⬇️ Instalar o AntiGravity

O primeiro passo concreto é o mais simples: baixar o AntiGravity a partir do site oficial — antigravity.google — e instalar como você instala qualquer outro aplicativo no seu computador.

🧭 Conceito principal

Não existe segredo aqui. É um programa que roda na sua máquina, como um editor de texto ou um navegador. Baixe da fonte oficial, abra o instalador e siga adiante.

  • •Acesse o site oficial (antigravity.google) — evite links de terceiros.
  • •Baixe a versão para o seu sistema (Windows, macOS ou Linux).
  • •Instale e abra — é o mesmo fluxo de qualquer app.

⚙️ Instalar o Node.js

O Node.js é o "motor" que deixa a IA criar e rodar projetos no nível do seu computador. Baixe de nodejs.org. Sem ele, a maioria dos apps simplesmente não roda localmente.

✓ Com Node.js instalado

  • ✓A IA consegue rodar o projeto na sua máquina
  • ✓Você instala dependências dos apps
  • ✓O localhost funciona normalmente

✗ Sem Node.js

  • ✗A maioria dos apps não inicia local
  • ✗Comandos de instalação falham
  • ✗Você trava logo no primeiro projeto

💡 Dica prática

Em nodejs.org, escolha a versão LTS (suporte de longo prazo): é a mais estável e a recomendada para a maioria das pessoas. Depois de instalar, basta reabrir o AntiGravity.

🐙 GitHub: repositórios

O GitHub é onde se guardam e versionam arquivos de projetos. Cada projeto vive num repositório (ou "repo"). É de lá que você importa código pronto — e para lá que você publica o seu mais tarde.

🌍 Repositório público

  • •Qualquer pessoa pode ver e importar
  • •Ótimo para templates e exemplos
  • •Aprende-se muito lendo código aberto

🔒 Repositório privado

  • •Só você (e quem convidar) acessa
  • •Ideal para projetos seus ou de clientes
  • •Mesma estrutura, visibilidade fechada

🟢 O botão verde "Code"

Em qualquer repositório, o botão verde "Code" revela a URL do repo. É essa URL que você vai colar no AntiGravity para importar o projeto. Guarde esse reflexo: "Code" → copiar URL.

📥 Importar projeto + localhost

Com a URL do repo em mãos, o ciclo é direto: você cola a URL e pede para o agente importar. Depois pede para rodar, e o projeto sobe num endereço local (localhost:porta) que só você vê.

1

Trazer o código

Cole a URL do repositório e peça ao agente para importar o projeto para a sua máquina.

2

Rodar

Peça para o agente iniciar o projeto. Ele instala o necessário e dá a partida.

3

Ver na tela

Ele sobe num endereço local, tipo localhost:3000. Abra no navegador e veja o app funcionando — só na sua máquina.

🌐 O que é "localhost"

"localhost" é o seu próprio computador funcionando como servidor. A porta (ex.: 3000, 5173) é só o "canal" onde o app está escutando. É um endereço privado: ninguém na internet vê, só você.

⌨️ O terminal / CLI

O terminal (também chamado de CLI, de "interface de linha de comando") é como você conversa com o computador por texto, em vez de cliques. Muitas ações — instalar coisas, rodar comandos, fazer builds — passam por ali.

exemplos de comandos no terminal
npm install      # instala o que o projeto precisa
npm run dev      # roda o projeto em modo de desenvolvimento
node --version   # mostra a versão do Node.js instalada

A boa notícia: no fluxo agentic, na maior parte do tempo é o agente quem digita esses comandos por você. Mas entender que o terminal existe — e que ele é só "o computador ouvindo texto" — tira o medo quando você precisa colar um comando manualmente.

💡 Dica prática

Não decore comandos. Quando precisar de um, peça ao agente: "que comando rodo para [objetivo]?". O terminal deixa de ser intimidante quando você o trata como mais um canal de conversa.

🌐 Browser control + autocorreção

Esse é o recurso que muda o jogo: o agente abre o navegador, clica em botões, confere o resultado na tela e se autocorrige — sem precisar de plugins. Ele "vê" o que aconteceu e ajusta sozinho.

Na prática, isso economiza muita ida e volta: em vez de você testar, copiar o erro, colar de volta e pedir conserto, o agente fecha esse loop sozinho. Você descreve, ele faz, verifica e corrige.

🔬 Exemplo prático

Vamos juntar tudo num gesto só. Importe um template de landing page de um repositório público e suba no localhost:

  • 1.Pegue a URL de um repo público com um template de landing simples.
  • 2.Cole no AntiGravity e peça para importar; depois peça para rodar.
  • 3.Abra o endereço local (ex.: localhost:3000) e veja o site na sua máquina.

Em poucos minutos, você sai do zero para um site rodando localmente — sem ter escrito uma linha à mão.

🏋️ Exercício

Deixe seu ambiente pronto de verdade. Faça o caminho completo:

  1. Instale o AntiGravity (de antigravity.google) e o Node.js (de nodejs.org, versão LTS).
  2. Importe um repositório público qualquer (um template simples serve).
  3. Rode no localhost e anote a porta que apareceu (ex.: 3000, 5173).

Se você viu o app abrir no navegador, seu ambiente está pronto para a Trilha 2 — onde construímos o primeiro app do zero.
