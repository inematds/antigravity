🧬 Master asset

A base do sistema é um master asset: um template de site lindo, bem feito uma única vez, e hospedado no GitHub. Em vez de começar do zero a cada cliente, você clona o repositório e apenas re-marca o que muda. O template é a fonte; cada cliente é um clone.

🧭 A ideia central

Pense no template como um molde: você desenha o bolo lindo uma vez e depois carimba cópias. O esforço difícil (layout, animações, responsividade) já está pago.

  • •Uma fonte única no GitHub — todo clone parte dela.
  • •Clonar o repo é quase instantâneo — nada de recriar a estrutura.
  • •Você só re-marca: troca o que identifica o cliente, mantém o resto.

🛠️ A Cloning Skill

Clonar à mão funciona uma ou duas vezes — depois você esquece um detalhe. A solução é uma skill: um prompt estruturado reutilizável que percorre sistematicamente os elementos a trocar, faz perguntas, pede os assets nas dimensões certas e só então implementa.

1

Percorre os elementos

A skill tem a lista do que muda e passa item por item — nada fica para trás.

2

Faz perguntas e pede assets

Pergunta o nome, a cor, os preços; pede o logo já dizendo a dimensão certa.

3

Implementa

Com tudo em mãos, ela aplica as trocas no clone — guiada e repetível.

💡 Dica prática

Você não precisa criar a skill do zero: ela está disponível na Biblioteca (Trilha 5). Comece dela e ajuste ao seu template.

🎨 Regra de design 70-20-10

Por que o esquema de cores é tão fácil de trocar? Porque o template segue a regra 70-20-10: 70% de uma cor dominante, 20% de uma cor secundária e 10% de um acento. Se tudo é destaque, nada é destaque — então o template já sabe onde a IA deve mexer na cor.

🛡️ Por que isso ajuda a clonar

Com uma proporção clara, trocar a marca é quase só mudar a cor dominante (o 70%). O layout permanece equilibrado, e o site novo já nasce profissional.

🗂️ Uma pasta por cliente

Quando você gerencia muitos clientes, o maior risco não é técnico — é mexer no site errado. A regra simples que evita isso: uma pasta por cliente no seu computador. Gestão de arquivos enxuta reduz confusão.

clientes/
├── dedetizadora-alfa/
│   └── site/        (clone re-marcado)
├── sorveteria-bela/
│   └── site/        (clone re-marcado)
└── advocacia-cardoso/
    └── site/        (clone re-marcado)

💡 Dica prática

Abra no editor só a pasta do cliente em que você vai trabalhar. Assim, é impossível a IA editar por engano o site de outro cliente.

🧾 Automatizar a coleta de dados

O próximo nível é tirar você do meio. Em vez de perguntar tudo no chat, crie um formulário próprio (logo + as 6 perguntas + upload de assets) que salva as respostas no Supabase via MCP. Depois você manda a IA "clonar a submissão mais recente" — sem nem olhar as respostas.

💡 Dica prática

Padronize os campos do formulário com os mesmos nomes que a Cloning Skill espera. Assim, a IA mapeia resposta → elemento sem ambiguidade.

♾️ Escalar

Com template-mestre + skill + formulário, a conta vira a seu favor: de 1 site para 100, para 1.000. O custo marginal por clone fica quase zero. E a cada rodada você refina a skill: rode uma vez, veja o que não trocou, ajuste, repita.

1

Rode uma vez

Gere um clone com a skill atual e olhe o resultado com atenção.

2

Ajuste o que não trocou

Achou um texto antigo ou uma cor esquecida? Acrescente esse item à skill.

3

Repita

A cada rodada a skill fica mais afiada — e os próximos clones saem perfeitos.

🔬 Exemplo prático

Pegue um template chamado "VOLTFLOW" (um template genérico de site de serviços) e gere duas versões para nichos diferentes, trocando apenas os 6 elementos:

Versão A — Dedetizadora

  • Logo da dedetizadora · cor dominante verde
  • Copy sobre controle de pragas · preços por visita
  • Avaliações de moradores · selos de segurança

Versão B — Sorveteria

  • Logo da sorveteria · cor dominante rosa
  • Copy sobre sabores · preços por tamanho
  • Avaliações de clientes · marcas de insumos

Mesmo template, mesma estrutura — dois sites que parecem feitos sob medida. Isso é a clonagem em ação.

🏋️ Exercício

Crie ou adapte a Cloning Skill e gere 2 variações do mesmo template para nichos diferentes:

  1. Escolha um template-mestre (ou use o seu da Trilha 2) e hospede no GitHub.
  2. Adapte a skill da Biblioteca para percorrer os 6 elementos do seu template.
  3. Gere 2 clones para nichos distintos (ex.: oficina e escritório de advocacia).
  4. Anote o que a skill esqueceu de trocar e refine-a.

Meta: na segunda variação, a skill já não esquece nada.
