🧬 Cloning Skill

A Cloning Skill é um protocolo que pega um template validado e o re-marca para um novo cliente ou nicho: troca logo, copy, marcas, preços, avaliações e a cor de acento — sem refazer a estrutura do zero.

✓ Quando usar

  • ✓Você já tem um template que funciona
  • ✓Precisa servir vários clientes parecidos
  • ✓Quer manter a estrutura e só trocar a marca

✗ Quando evitar

  • ✗O novo projeto tem estrutura diferente
  • ✗Você ainda não validou o template original
  • ✗O conteúdo muda mais do que a marca
cloning-protocol.md

protocolo de clonagem / rebrand

⬇️ Baixar skill
uso rápido
Use a Cloning Skill neste template. Re-marque para o cliente abaixo,
trocando: logo, copy, nome de marca, preços, avaliações e cor de acento.
Mantenha a estrutura intacta.

Cliente: [nome] · nicho: [nicho] · cor: [hex]

🔄 Transformer Skill

A Transformer Skill converte um site feito num framework em um único HTML self-contained — tudo embutido (CSS e JS inline) — pronto para colar dentro de plataformas como GoHighLevel ou WordPress.

📦 Entregar onde o cliente já está

Muitos clientes já têm o site num GoHighLevel ou WordPress. Em vez de migrar tudo, você transforma sua página num bloco único e cola lá. Um arquivo, zero dependências externas.

converting-to-gohighlevel.md

framework → HTML self-contained

⬇️ Baixar skill
uso rápido
Use a Transformer Skill: converta este projeto num único HTML
self-contained (CSS e JS inline, sem dependências externas),
pronto para colar dentro do GoHighLevel/WordPress.

🐞 Ultimate Debugger Prompt

O Ultimate Debugger Prompt é um template estruturado (em XML) para debugar erros com IA de forma disciplinada: o modelo gera predições, investiga, acha a causa-raiz e dá instruções passo a passo. É uma ferramenta open-source de terceiros.

1

Predições

A IA lista hipóteses do que pode estar causando o erro antes de investigar.

2

Investigação → causa-raiz

Verifica cada hipótese e converge para a origem real do problema.

3

Instruções passo a passo

Entrega o conserto detalhado, na ordem, pronto para aplicar.

ultimate-debugger-prompt.md

template XML de debug · open-source

⬇️ Baixar prompt
uso rápido
<debug>
  <erro>[cole a mensagem de erro]</erro>
  <contexto>[o que você estava fazendo]</contexto>
  <tarefa>Liste predições, investigue, ache a causa-raiz e me dê
  instruções passo a passo para corrigir.</tarefa>
</debug>

🔌 MCP Config

O MCP Config é uma configuração pronta de servidores MCP — n8n, Notion, Pinecone, Supabase, Vercel e Fireflies — com placeholders de chave. É o que liga seu agente às ferramentas externas.

MCPPara quê
n8ndisparar e ler automações
Notionler/escrever em páginas e bases
Pineconebusca vetorial / memória
Supabasebanco de dados e auth
Verceldeploy e domínios
Firefliestranscrições de reuniões

🔑 Cuidado com as chaves

O arquivo vem com placeholders (ex.: SUA_CHAVE_AQUI). Troque pelos seus valores e nunca suba chaves reais para repositórios públicos.

mcp-config.json

6 MCPs com placeholders de chave

⬇️ Baixar config
uso rápido
Conecte o MCP do [Supabase] usando esta config. Me oriente passo a passo
onde colar a configuração e onde substituir a chave (placeholder).
Depois, confirme que a conexão está funcionando.

✅ QA Pre-Flight Checklist

O QA Pre-Flight Checklist é um protocolo de QA e stress test por fases, que classifica cada achado em três níveis de severidade — Crítico, Aviso e OK. É a sua checagem antes de "decolar" (publicar).

🔴 Crítico

trava o uso · não pode publicar

🟡 Aviso

funciona, mas melhora depois

🟢 OK

passou no teste

qa-pre-flight-checklist.md

QA por fases · Crítico/Aviso/OK

⬇️ Baixar checklist
uso rápido
Rode o QA Pre-Flight Checklist neste projeto. Faça o stress test por fases
e classifique cada achado como Crítico, Aviso ou OK. Liste primeiro os
Críticos e diga se posso publicar.

🔬 Exemplo prático

Vamos colocar a biblioteca para rodar: baixe a Cloning Skill e use-a para trocar o nicho de um template que você já tenha.

  1. Baixe cloning-protocol.md no tópico 1.
  2. Aponte a skill para um template seu validado.
  3. Peça para re-marcar para um novo nicho (ex.: de "clínica" para "academia"): logo, copy, preços, avaliações e cor.
  4. Confira que a estrutura ficou intacta e só a marca mudou.

Em minutos você tem uma segunda versão pronta — esse é o poder de reaproveitar artefatos da biblioteca.

🏋️ Exercício

Escolha 1 skill desta biblioteca e adapte ao seu projeto:

  1. Baixe o artefato (skill, config ou prompt).
  2. Edite-o para o seu contexto — troque placeholders, nichos, comandos ou critérios.
  3. Salve a sua versão com um nome próprio e teste no projeto.

Dica: comece pela claude.md (tópico 5) — ela serve de base para todas as outras etapas do CODA.
